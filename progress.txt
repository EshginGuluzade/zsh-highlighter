## Codebase Patterns
- Rust binary with zero external dependencies (no regex crate, no serde, no syntect)
- All tokenizer tests use byte offsets, not char offsets
- region_highlight format: "{start} {end} {style}" with 0-indexed byte offsets
- $_ZH_CMDS env var is newline-separated list of known commands
- When $_ZH_CMDS is empty, all commands default to valid (fg=green,bold)
- Reserved words list: if then else elif fi for in while until do done case esac function select repeat time
- { } are tokenized as Operator tokens initially, then restyled to ReservedWord by the command position pass
- [[ ]] are tokenized as Operator tokens initially, then restyled to ReservedWord by the command position pass
- ( ) remain Operator tokens always

## Tokenizer Design
- The tokenizer MUST be a single-pass character-by-character state machine
- States: Default, InWord, InSingleQuote, InDoubleQuote, InAnsiCQuote, InBacktick, InComment
- In Default state: skip whitespace, detect string openers, detect operators, detect comment start, otherwise start a Word
- '#' only starts a comment when it would begin a new token (preceded by whitespace or at position 0) — NOT mid-word
- $' (dollar + single quote) starts an ANSI-C string — dollar followed by anything else is just part of a Word
- Line continuation (backslash + newline) is treated as whitespace and skipped
- All Word tokens initially have command_position: false — command position is determined in a second pass (US-003)
- The command position pass walks the token list with a boolean flag 'expect_command' (starts true). After operators | || && ; ;; ( and after reserved words then/else/elif/do/time/!, set expect_command=true. After for/case/select/in/[[, set expect_command=false.

## US-002 — Complete tokenizer (complete)
- Single-pass state machine with 7 states: Default, InWord, InSingleQuote, InDoubleQuote, InAnsiCQuote, InBacktick, InComment
- match_operator() helper does greedy operator matching (<<< before <<, || before |, etc.)
- In Default state: whitespace skipped, line continuations skipped, then detect comment/#, ANSI-C $', quotes, operators, otherwise start Word
- In InWord state: line continuation BREAKS the word (emits word, returns to Default) — not just skipped
- String openers (', ", `) break a word mid-token and start a new String token (adjacency like echo"hello")
- Operators break words too (adjacency like echo>file)
- # only starts a comment when at position 0 or preceded by whitespace — checked via bytes[i-1]
- $' starts ANSI-C string only when $ is immediately followed by ' — otherwise $ is just part of a word
- Single [ and ] are NOT operators — they're part of words. Only [[ and ]] are operators.
- Unterminated strings/backticks extend to end of input
- End-of-input flush: after the loop, check state and emit any pending token
- The loop also has internal end-of-input checks at the start of each state (InWord, InSingleQuote, etc.) that emit and break
- 47 tests total covering all acceptance criteria

## US-003 — Command position detection and reserved words (complete)
- Implemented as mark_command_positions() second pass in tokenizer.rs
- Two flags: expect_command (bool, starts true) and expect_in_keyword (bool, starts false)
- expect_in_keyword is needed because `in` appears as a reserved word AFTER a non-command word (e.g., `for i in`) — can't detect it with expect_command alone
- After for/case/select: expect_command=false, expect_in_keyword=true; the `in` check runs before the command position check
- expect_in_keyword persists through non-command words (don't clear it until `in` is found or an operator resets state)
- After if/while/until/then/else/elif/do/time: expect_command=true
- After fi/done/esac: expect_command=true (block terminators, next thing is usually operator or new command)
- After function/repeat: expect_command=false
- ! in command position becomes ReservedWord with expect_command=true (next word is still command)
- { } operators restyled to ReservedWord; { sets expect_command=true
- [[ ]] operators restyled to ReservedWord; [[ sets expect_command=false (conditional args follow)
- |& and & also set expect_command=true
- Redirects (>, >>, <, <<, <<<) and ) do NOT change expect_command
- Removed #[allow(dead_code)] from TokenType enum since all variants are now constructed
- Changed mark_command_positions signature from &mut Vec<Token> to &mut [Token] per clippy
- 27 new tests for command position (74 total)

## US-004 — Classifier: map tokens to region_highlight styles (complete)
- classify() takes tokens slice, input string, and known_commands HashSet
- Added `input: &str` parameter to classify() so it can extract command words via &input[token.start..token.end]
- Style mapping: Word(cmd=true)+known → fg=green,bold; Word(cmd=true)+unknown → fg=red,underline; Word(cmd=false) → skip (no highlight); ReservedWord → fg=yellow,bold; String → fg=yellow; Comment → fg=8; Operator → fg=cyan
- When known_commands is empty, all command-position words default to fg=green,bold (safe default)
- Arguments (Word with command_position=false) produce NO highlight entry — they are simply skipped
- Updated main.rs to pass `input` as second arg to classify()
- 13 new tests: unit tests for each token type mapping + 5 end-to-end tests using tokenize+mark_command_positions+classify
- 87 total tests passing

## US-005 — Zsh plugin: core highlighting hook (complete)
- plugin/zsh-highlighter.zsh is 37 lines (under 50 limit)
- Binary location: checks ${0:A:h}/../bin/zsh-highlighter first, falls back to ${commands[zsh-highlighter]} (zsh hash table lookup, equivalent to command -v but no fork)
- $_ZH_CMDS built from ${(k)commands} ${(k)builtins} ${(k)aliases} ${(k)functions} joined with newlines via ${(pj:\n:)_cmds}
- _zh_rebuild_cmds() extracted as reusable function (needed by US-006 for zh-reload and chpwd)
- _zh_highlight guard clauses: empty BUFFER, unchanged BUFFER, >10000 bytes
- Binary stderr suppressed with 2>/dev/null; non-zero exit clears region_highlight
- region_highlight populated via "${(@f)_zh_output}" which splits on newlines
- If binary not found at load time, plugin returns silently (return 0)
- zsh -n syntax check passes
